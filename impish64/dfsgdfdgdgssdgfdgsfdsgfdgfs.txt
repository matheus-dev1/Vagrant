A maquina virtual "Ansible" com Ansible instalado, vai ter um arquivo playook.yml com os passos para configurar a maquiva virtual "mysqlserver".

A conexao da maquina virtual "ansible" com a "mysqlserver" e feita via ssh.

A maquina virtual que envia dados, os comandos, no meu caso a "ansible" deveria ter uma chave privada e a que esta acessando, no meu caso "mysqlserver", deveria ter a chave publica.

Para executar um comando ansible dentro de uma maquina virtual , no usamos o comando ansible-playbook -i [diretorio do seu arquio hosts] [diretorio do seu arquivo playbook.yml]

As chaves tem uma validade, e as vezes é necessarios recria-las.

Se voce criou uma senha para sua chave ssh, voce deve configura-la no seu arquivo hosts.

O comando "vagrant global-status" me mostra todos os ambientes da minha maquina(host).
Obs: O parametro --prune vai me mostrar apenas os ambientes atualizados e não algum que pode estar em cache.
Obs 2: Eu posso usar qualquer comando em uma maquina se eu tiver o id dela, onde pode ser obtido no comando "vagrant global-status"
Obs 3: Uma maquina só vai aparecer nesta lista se ela foi criada uma vez.


O comando "vagrant box" me permite executar comando referente as boxes do vagrant, como por exemplo listar todos os boxes instalados na minha maquina com o comando "vagrant box list".

Lembrete, o Vagrant trabalha com Virtualização.

Com o vagrant controlamos neste exemplo virtual box atraves de codigo.

Container
Ao invés de trabalharmos com um hipervisor, utilizamos o recurso de Container Engine representando pelo Docker, o qual depende do Linux. Logo, não temos simulações de outros sistemas operacionais para rodar as ferramentas.

Ou seja, se já temos uma biblioteca de dados carregada no sistema operacional, não é preciso que os programas configurados possuam suas própria, pois se conecta diretamente à fonte através dos containers, tornando-os mais leves e rápidos.
